# Production Deployment System

Эта система обеспечивает постоянную работу проекта с автоматическим перезапуском и обновлением.

## 📁 Скрипты

### Основные скрипты:

| Скрипт | Описание |
|--------|----------|
| `start-production.bat` | 🚀 **Главный скрипт** - запускает всю систему |
| `build-prod.bat` | Собирает проект для production |
| `run-prod.bat` | Запускает сервер с автоматическим перезапуском |
| `watch-and-restart.bat` | Следит за изменениями файлов и пересобирает проект |
| `add-to-startup.bat` | Добавляет проект в автозагрузку Windows |

## 🚀 Быстрый старт

### Вариант 1: Ручной запуск
```batch
start-production.bat
```

### Вариант 2: Автозагрузка при старте Windows
```batch
add-to-startup.bat
```

## 📋 Как это работает

1. **Production сборка**: `npm run build` создает оптимизированную версию
2. **Сервер**: Запускается на `http://localhost:3000`
3. **Автоперезапуск**: Если сервер падает, перезапускается через 5 секунд
4. **Автообновление**: Каждые 10 секунд проверяет изменения и пересобирает
5. **Непрерывная работа**: Работает даже в спящем режиме компьютера

## 🔧 Преимущества

✅ **Непрерывная работа** - сервер всегда запущен  
✅ **Автоперезапуск** - автоматически восстанавливается после сбоев  
✅ **Автообновление** - применяет изменения без ручного вмешательства  
✅ **Автозагрузка** - запускается вместе с Windows  
✅ **Production режим** - оптимизированная производительность  

## 📊 Мониторинг

Сервер доступен по адресу: **http://localhost:3000**

Логи отображаются в консоли с временными метками.

## ⚠️ Важно

- Не закрывайте консоль с запущенным скриптом
- Для остановки используйте Ctrl+C
- При изменении файлов сервер автоматически пересобирается
- Проект работает в production режиме (оптимизирован)

## 🛑 Остановка

Нажмите `Ctrl+C` в консоли с запущенным скриптом.
